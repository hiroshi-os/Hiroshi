Hiroshi uses an advanced local **Reciprocal Rank Fusion (RRF)** retrieval routine. Instead of relying purely on semantic vector matches (which can miss specific code strings) or basic keyword parsing (which misses conceptual synonyms), it combines both approaches concurrently.

### 📊 The 70/30 Fusion Algorithm

When your agent invokes a memory retrieval check, the engine triggers two asynchronous search threads simultaneously across your hard-locked SQLite database:

$$\text{Final RRF Score} = (0.70 \times \text{Vector Rank Score}) + (0.30 \times \text{FTS5 Keyword Rank Score})$$

```text
                     ┌──> Async Vector Search (70% Weight) ──┐
[ Memory Query ] ────┤                                       ├──> [ Reciprocal Rank Fusion ] ──> Top-N Clean Context
                     └──> Async FTS5 Match (30% Weight) ─────┘
```

This ensures maximum recall reliability. If your local embedding endpoint drops offline or experiences an unusual network hiccup, the search algorithm falls back to pure keyword FTS5 scanning automatically, keeping your memory layer functional.
