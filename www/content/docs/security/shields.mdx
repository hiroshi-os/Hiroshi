title: Interface Security & Path Jails
description: Verifying input types and directory perimeters at compile-time.
---

Hiroshi implements an absolute, zero-trust system protection system at its execution borders to defend your computer from parameter injection or relative path breakouts.

### 🛡️ 1. Rigid Frame Validation
All data frames entering the WebSocket gateway or platform channels are parsed via a hard-locked `GatewayFrame` validator utilizing `#[serde(deny_unknown_fields)]`. 

If an incoming JSON payload contains unexpected keys, unknown structures, or loose object parameters often seen in exploit payloads, the engine discards the stream instantly at the network boundary before allocating an internal conversation thread.

### 🚫 2. Canonical Path Containment (Jail Verification)
When an agent or script tries to manipulate files within your workspace, the sandbox forces target resolution checks using strict system tools:

```rust
// Under-the-hood security baseline
dunce::canonicalize(target_path)
```

The system evaluates the canonical physical address on your storage drive. If a symbolic link, terminal shortcut, or relative path breakout string (like `../../etc/`) points outside the absolute bounds of `~/.hiroshi/workspace/`, the application drops the process instantly, returning an explicit `PermissionDenied` exception block.
