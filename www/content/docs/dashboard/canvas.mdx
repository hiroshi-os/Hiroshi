title: The Visual Canvas Dashboard
description: Monitor running background agents and revoke permissions using the local web cockpit.
---

Hiroshi hosts a local desktop web interface at `http://127.0.0.1:8080/`, giving you a visual overview of active background agents and running processes.

---

## 🖥️ Left Panel: The Workspace Cockpit

The cockpit panel focuses on agent status tracking, execution metrics, and security controls.

### 🎭 Multi-Agent Theater
*   **Active Focus Indicators**: Displays cards for all registered agents. When a handoff happens, the interface flashes its border colors and updates the focus layout to highlight the agent now in control.
*   **Animation States**: Shows real-time thinking states as text completes in the background.

### 📊 Live Telemetry
*   **Token Metrics**: Tracks model completion speed (tokens per second) and loop response latencies.
*   **System Monitors**: Simple indicators tracking processor loads and memory consumption.

### 🔐 Runtime Permission Hub
An interactive security control center.
*   **Interactive Checkboxes**: Lists all loaded skills (including custom scripts and reflected MCP tools) next to toggling checkboxes.
*   **Live Privilege Revocation**: Uncheck any box to instantly disable that capability. The active agent will lose permission to execute that tool, allowing you to secure files without editing configuration text files.

---

## 🖼️ Right Panel: The Multi-Media Canvas

The canvas panel displays the outputs of your sandboxed workspace processes.

### 📸 Visual Viewport
*   **Screenshot Streams**: When browser automation tools (like Playwright) capture viewport images, they are sent to the cockpit as Base64-encoded screenshots. The canvas decodes and renders these images on a simulated display, letting you watch web scraping or UI test runs live.

### 📜 Live Console Streams
*   **Process Output Loggers**: Displays compilation logs and raw terminal output logs from command executions inside the sandbox.
*   **Telemetry Logs**: A real-time viewport into `hiroshi.log`, capturing agent handoff paths and tool call metrics as they happen.
