Welcome to the **Hiroshi Core** developer guide. Hiroshi operates as a secure, background-autonomous edge daemon designed to bridge local processing power with developer workflows.

### System Metrics
- **Runtime Idle Memory:** `< 40 MB` RAM
- **Routing Loop Overhead:** `< 5ms` execution latency
- **Network Dependency:** 100% offline-capable via native local implementations

### Structural Overview
Hiroshi's architecture is built around dynamic multi-agent collaboration, secure execution sandboxes, local retrieval-augmented generation (RAG), and zero-overhead process extensions:

- **🚀 Getting Started**: Learn how to build and launch the Hiroshi daemon.
- **🧠 Architecture**: Deep dive into the internal agent handoff loops, database schema, and process shields.
- **🔌 Extensibility**: Create custom polyglot skills or integrate third-party Model Context Protocol (MCP) servers.
