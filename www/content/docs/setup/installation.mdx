---
title: Installing from Source
description: Build the zero-dependency, static Hiroshi executable from source.
---

Hiroshi compiles down to a single, zero-dependency release binary that embeds all core business logic, RAG memory engines, and web dashboard assets inside it.

### Prerequisites

To compile Hiroshi, ensure you have the Rust toolchain installed:

- **Rust / Cargo**: 1.75+ (tokio, async/await support)
- **C/C++ Compiler**: Required for compiling the local SQLite SQLite3 libraries.

### Building from Source

1. Clone or enter your workspace directory:
   ```bash
   cd Hiroshi
   ```

2. Compile the binary in release mode to bundle all web dashboard assets statically:
   ```bash
   cargo build --release
   ```

3. The compiled zero-dependency binary will be generated at:
   - **Windows**: `target/release/hiroshi.exe`
   - **Linux/macOS**: `target/release/hiroshi`

### Launching the Daemon

Simply execute the compiled binary with a subcommand:
```bash
# Launch interactive agent chat prompt
./target/release/hiroshi agent

# Launch background service daemon
./target/release/hiroshi daemon
```
If this is the first execution on your host machine, the boot loader will automatically intercept your terminal and redirect you into the **Interactive Onboarding Wizard**.

---

### Custom Compilation Features

Hiroshi supports modular feature flags that allow you to customize the binary footprint and compile only what you need:

| Feature Flag | Description | Added Dependencies |
| :--- | :--- | :--- |
| `gateway-ui` | Embeds the local web dashboard, server, and assets | Axum, Tower-HTTP |
| `channel-telegram` | Enables the Telegram gateway connector | None |
| `channel-discord` | Enables the Discord gateway connector | Tokio-Tungstenite |
| `channel-slack` | Enables the Slack gateway connector | Tokio-Tungstenite |
| `kernel-only` | Compiles *only* the core string engine, SQLite storage, and sandbox | None |

#### Compiling a Lean Headless Kernel

If you only want a lightweight terminal agent without any UI, background servers, or network listening suites:

```bash
cargo build --release --no-default-features --features kernel-only
```
Compiling with `kernel-only` drops the final release binary size under **< 7 MB** with an idle memory usage of under **< 5 MB RAM**.
