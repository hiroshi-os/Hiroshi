---
title: System Diagnostics (Doctor)
description: Run self-diagnostic environment health checks on your Hiroshi node.
---

If you run into issues connecting to your LLM provider, saving files inside the sandbox, or encountering database crashes, you can run Hiroshi's diagnostic suite to verify the node's environment integrity.

Simply trigger the `doctor` command:

```bash
hiroshi doctor
```

### 📋 Diagnostic Checks Performed

Hiroshi's doctor automatically runs four diagnostic validation phases:

1.  **Ollama Connectivity Check** - Tests TCP connections to the configured Ollama host (`config.ollama.host`) to verify the local LLM model service is active and responsive.
2.  **Workspace Permissions Check** - Performs temporary write and read operations inside your designated sandbox folder (`~/.hiroshi/workspace/`) to verify proper filesystem access.
3.  **SQLite DB Integrity Scan** - Executes a database check (`PRAGMA integrity_check`) to scan for transaction failures or WAL database file corruption.
4.  **Environment Configuration Check** - Summarizes active engine configurations, embedding models, workspace directories, and Tailscale overlay statuses.

---

### 📊 Understanding the Scorecard Output

Upon completion, `hiroshi doctor` prints a health scorecard:

```text
==================================================
          HIROSHI SYSTEM DIAGNOSTICS            
==================================================
Checking Ollama Connectivity... [OK] Connected to Ollama at http://127.0.0.1:11434
Checking Workspace Permissions... [OK] Read/Write tests passed.
Checking SQLite DB Integrity... [OK] Integrity check passed.

Configured Components:
  - Target Model:      qwen2.5-coder:1.5b
  - Embedding Model:   nomic-embed-text
  - Log Level:         info
  - Workspace Path:    C:\Users\User\.hiroshi/workspace
  - Database Path:     C:\Users\User\.hiroshi\hiroshi.db
  - Tailscale Overlay: Disabled
==================================================
          ALL DIAGNOSTIC CHECKS PASSED          
```

If any check returns `[FAIL]`, the tool exits with status code `1`, indicating the specific error detail to help you resolve system permission mismatches or service connection drops.
