title: CLI Subcommands & Operation Modes
description: Navigating the dual-architecture interface of Hiroshi.
---

Hiroshi splits its operational profile into two distinct, highly optimized binaries using a native command-line parser. 

### 💬 1. Interactive Shell Mode (`hiroshi agent`)
To interact with a single agent persona directly over a high-speed, local offline console interface, trigger the `agent` subcommand:

```bash
hiroshi agent
```

* **Slash Commands**: Type `/help`, `/clear` (wipes the active short-term memory canvas context), `/agents`, `/skills`, or `/exit` (ends your session).
* **Environment Mapping**: Automatically attaches custom user identity properties from `AGENTS.md` directly into the terminal stream.

### 🛡️ 2. Headless Daemon Mode (`hiroshi daemon`)
To boot Hiroshi as an always-on automated assistant tracking long-running event queues, activate the daemon:

```bash
hiroshi daemon
```

This starts the long-running background scheduler engine, opens your local Axum web server, hooks into your verified messaging channels (Telegram, Discord, Slack), and enforces a strict cross-process advisory lock file at `~/.hiroshi/.gateway.lock` using filesystem-level primitives to completely eliminate concurrent database write corruption.
