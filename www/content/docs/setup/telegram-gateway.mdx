---
title: Connecting Telegram Bots
description: Connect Hiroshi to Telegram to prompt agents and execute sandboxed code changes remotely.
---

You can connect Hiroshi to a Telegram bot, allowing you to prompt agents and execute sandboxed code changes remotely from your mobile device.

## 1. Create a Telegram Bot

1. Find [@BotFather](https://t.me/botfather) on Telegram.
2. Send the `/newbot` command and follow instructions to name the bot.
3. Retrieve your **HTTP API Token** (e.g. `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`).

## 2. Retrieve your Telegram User ID

For security, Hiroshi drops any incoming traffic that does not come from your whitelisted Telegram account.

1. Find [@userinfobot](https://t.me/userinfobot) on Telegram.
2. Send a message to retrieve your numerical User ID (e.g. `987654321`).

## 3. Configuration Setup (`config.toml`)

Enable the gateway and add your credentials in `~/.hiroshi/config.toml`:

```toml
[telegram]
enabled = true
token = "123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ" # Your bot token
allowed_user_ids = [987654321]                 # Your numerical User ID
```

## 4. Run & Chat Remotely

1. Start the daemon with `cargo run`. You will see:
   ```text
   [Telegram] Gateway listening service started.
   ```
2. Open your Telegram bot chat and send a message.
3. Hiroshi will print `Thinking...` and stream the responses back to Telegram using Markdown formatting.
4. **Handoffs** work exactly like the terminal: the daemon will detect routing tags and swap active agent prompts automatically in the background.
