---
title: Tailscale Remote Access
description: Securely access your Hiroshi dashboard from anywhere using private Tailscale overlays.
---

Hiroshi integrates natively with your local **Tailscale** daemon. When enabled, Hiroshi binds its local Web UI cockpit and WebSocket gateways directly to your private Tailnet IPv4 address. This makes your agent safely accessible from your mobile phone or laptop on the go without exposing port `8080` to the public internet.

---

### ⚙️ Enabling Tailscale Overlay

To enable Tailscale integration, open your `~/.hiroshi/config.toml` file and add or configure the `[tailscale]` block:

```toml
[tailscale]
enabled = true
# Optional fallback IP address if Tailscale resolution is offline
interface_fallback = "127.0.0.1"
```

### 🚀 How it Works

1. On boot, the Hiroshi daemon checks if `tailscale.enabled` is set to `true`.
2. Hiroshi executes the system command `tailscale ip -4` to dynamically query your active Tailnet IPv4 address.
3. Upon success, the Axum Web Server binds directly to `<tailnet_ip>:8080` instead of localhost.
4. You can now open `http://<your-tailscale-ip>:8080/` on any device logged into your Tailscale network!

### 🔍 Troubleshooting

If the daemon prints a resolution warning on start, verify that:
*   The Tailscale client is running on the host machine.
*   The `tailscale` CLI command is accessible in your system's PATH.
*   You can manually run `tailscale ip -4` in your terminal to see a valid IPv4 address.
