---
title: Custom Polyglot Skills
description: Adding execution extensions to your agent without compiling core source code.
---

Hiroshi supports dynamic execution hooks natively. To register a capability, drop a folder into `~/.hiroshi/skills/` containing:

1. `SKILL.md` — Holds the tool schema definition used by the LLM prompt layer.
2. An executable script matching the name (e.g., `web_scrape.py` or `compile.sh`).

> ⚠️ **Execution Shield Guard:** Every spawned tool process is strictly monitored by a hard 10-second timeout circuit breaker thread to protect host processor load limits.

### Designing a Custom Skill Folder

Consider creating a custom skill `image_compressor`:

```text
~/.hiroshi/skills/image_compressor/
├── SKILL.md
└── image_compressor.py
```

#### The `SKILL.md` Template

Provide structured YAML frontmatter for description and arguments metadata:

```yaml
---
name: image_compressor
description: "Compresses local png/jpg image assets inside the workspace."
schema: '{ "image_path": "string", "quality": "number" }'
---
# Image Compressor Skill
Reduces image sizes using python pillow libraries.
```

#### The Python Handler

Your handler should accept JSON parameters on `stdin` and print final execution outputs to `stdout`:

```python
import sys, json, os

def main():
    # Read arguments parsed by Hiroshi
    args = json.loads(sys.stdin.read())
    path = args.get("image_path")
    quality = args.get("quality", 85)
    
    # Run logic safely
    if os.path.exists(path):
        # perform compression...
        print(f"Compressed {path} successfully.")
    else:
        print(f"Error: Image {path} not found.")

if __name__ == '__main__':
    main()
```
Now, Hiroshi automatically registers this skill on boot and adds it to the prompt schema of active agents containing permissions to invoke it.
